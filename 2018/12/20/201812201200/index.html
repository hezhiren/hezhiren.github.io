SpringBoot + Freemarker + LayUI框架搭建

由于一直做后台开发，对于前端的框架及js，css语法不是很熟悉，因此，搭建这个小项目用于学习前端知识。
此项目使用SpringBoot、Freemarker，以及LayUI等框架，是一个用于个人财务管理的一个后台系统。
学习目标：

  • 掌握js，css语法，能够熟练使用前端框架
  • 后台系统模拟正式项目，模块化开发，有单点登陆模块、日志模块、权限管理等，旨在系统的学习常规项目的构成元素及原理。

一、准备工作

首先搭建spring项目的框架，使用https://start.spring.io/ ,可以快速搭建一个简单的项目，并且可以添加常用的依赖。项目的搭建就不再次赘述，下面重点是依赖的添加以及layui和freemarker的配置。

首先做一下数据库的配置：

# datasource config
spring.datasource.driver-class-name=com.mysql.jdbc.Driver
spring.datasource.url=jdbc:mysql//127.0.0.1:3306/finance_manager?userSSL=false&charset=utf-8
spring.datasource.username=root
spring.datasource.password=root

接着是freemarker的配置

# FTL配置
spring.freemarker.suffix=.ftl
spring.freemarker.charset=UTF-8
spring.freemarker.template-loader-path=classpath:/templates/
spring.freemarker.cache=false
multipart.maxFileSize=60MB
multipart.maxRequestSize=60MB

# FTL中引用js和css路径配置
spring.freemarker.request-context-attribute=base
spring.freemarker.expose-request-attributes=true
spring.freemarker.expose-session-attributes=true
spring.freemarker.expose-spring-macro-helpers=true

静态资源统一放在resource下的static文件夹下，ftl文件放在templates文件夹下，至此，简单的配置就完成了。

至于layui相关的文件，如何放置以及文件的目录之间的关系在官方文档有介绍，这里就不多说了，我是放在static/layui下的。

在此处需要注意的问题，${base.contextPath}中的base是在properties配置文件中的spring.freemarker.request-context-attribute，此上下文可用于获取项目根目录。
在此处遇到很坑的一个问题就是我是用IDEA创建的项目，默认是有webapp文件夹的，但是项目不需要，我就把文件夹中的东西给删了，但是在Project-Structure界面查看module，发现项目配置了web.xml文件路径，但是此文件被我删了，导致页面显示时，找不到js和css文件，把这个配置的路径删除即可正常显示。
